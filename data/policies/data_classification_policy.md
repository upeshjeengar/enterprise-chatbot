# Data Classification Policy (v1.9)

**Owner:** Data Protection Office
**Applies to:** Classification of all AcmeCorp data assets.

## Section 1. Classification Levels
- **Public** — may be freely shared. Example: marketing brochures.
- **Internal** — for employees; low harm if leaked. Example: org charts.
- **Confidential** — business-sensitive; requires NDA to share externally. Example: contracts, roadmaps, analytics dashboards.
- **Restricted** — highest sensitivity; regulated. Example: customer PII, payment data, credentials.

## Section 2. Handling Requirements
| Level | External sharing | Access approval |
|---|---|---|
| Public | Allowed | None |
| Internal | Manager approval | Manager |
| Confidential | NDA + Legal | Security |
| Restricted | DPO + Legal + encryption | Data Protection Officer |

## Section 3. Agent Guidance
- The classification of the data involved determines the required approver. When in doubt, the agent escalates to the stricter tier and asks a human.
