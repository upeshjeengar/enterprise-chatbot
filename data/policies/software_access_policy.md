# Software & System Access Policy (v2.2)

**Owner:** IT & Identity Management
**Applies to:** Provisioning of software licenses and system access for employees and vendors.

## Section 1. Access Request Basics
- **Section 1.1** — All access requests must name a business owner and an access duration.
- **Section 1.2** — External vendors receive time-boxed access only (default 90 days, renewable with re-approval).

## Section 2. GitHub / Source Code Access
- **Section 2.1** — Repository access for vendors requires Security review and Business Owner approval.
- **Section 2.2** — Write access to production branches is never auto-granted.

## Section 3. Provisioning Rules
- **Section 3.1** — The agent may **create** an access request ticket in the IAM system.
- **Section 3.2** — The agent must **not** activate/provision access until all required approvals are complete.

## Section 4. Deprovisioning
- Access is automatically flagged for review at the end of the access duration.
